§ 5 — Managing Bias & Variance trade-off

Managing the trade-off

L1/L2
L1/L2 Regularisation
L2 shrinks coefficients toward zero.
L1 can kill some coefficients entirely.
Variance ↓ Bias ↑
BAG
Bagging
Each individual tree overfits, but their mistakes are somewhat independent.
Variance ↓ Bias same
+n
More Data
Gives the model less room to hallucinate if the model/architecture is correctly selected.
Variance ↓ Bias —
BOOST
Boosting
Start with a very simple (high bias) model.
Look at what it got wrong.
Train the next model to fix those errors
Repeat.
Variance ↑ Bias ↓
CV
Cross-Validation
If a model overall suits well, it won't provide a too biased model or with high variance.
Variance balance Bias balance
ARCH
Model Choice
A linear regression has high bias and low variance.
A deep neural network can have low bias and high variance.
Variance depends Bias depends
Rule: if Bias² dominates → richer model / less regularisation. If Var dominates → regularise, add data, or aggregate.
