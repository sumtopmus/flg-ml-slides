Hyperparameter Answers
FFN size vs hidden size?
Original transformer: 4× hidden size.
With SwiGLU (two matrices): ~2.67× to keep total params the same.
Rule: ffn_dim = (8/3) * d_model, rounded to a nice multiple.
How many heads?
Typically d_model / 64 — each head gets 64 dims.
Yes, num_heads must divide d_model exactly (head_dim = d_model / num_heads).
Modern models often use GQA: fewer KV heads than Q heads to save memory.
Vocab size?
Typically 32k–100k tokens (GPT-2: 50k, LLaMA: 32k, Gemma: 256k).
Larger vocab → shorter sequences, but bigger embedding table.
Balance: ~100 tokens per parameter in the embedding layer.
Do people regularize huge LMs?
Almost no dropout — large models don't overfit in the classical sense (they're undertrained).
Still use: weight decay (~0.1) and gradient clipping (~1.0).
Deep vs wide?
Both matter, but depth scales better for reasoning and generalisation.
Chinchilla scaling laws: balance model size with compute budget.
The ratio d_model / n_layers is called the aspect ratio — wider-per-layer vs more layers. Typical value: ~128.
