Sparse & Sliding Window Attention
Problem
Full attention is O(n²)
Every token attends to every other token.
For long sequences (100k+ tokens) this becomes prohibitively expensive in both compute and memory.
complexity: O(n²) — full matrix
Solutions
Sliding Window & Sparse Attention
Sliding window: each token attends only to the W nearest neighbours. Information propagates via depth over layers.

Sparse patterns: combine local window + a few global tokens (e.g. [CLS], task tokens) that attend everywhere.
complexity: O(n · W) — window size W ≪ n
Used in: Mistral (sliding window), Longformer, BigBird (sparse + global tokens). Enables context lengths of 128k–1M tokens that would be impossible with full attention.
