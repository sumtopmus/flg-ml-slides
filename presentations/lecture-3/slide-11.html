Fixing the Softmax
Attention softmax is already safe
Before computing attention scores, we apply QK LayerNorm — normalise Q and K vectors directly.

This bounds the dot products going into softmax, so the partition function Z can't explode in the attention layer.
scores = norm(Q) · norm(K)ᵀ / √d
Soft-capping the output logits
The final output logits (before the vocabulary softmax) can still grow large. We clamp them smoothly with tanh:
logits = soft_cap · tanh(logits / soft_cap)
Values are squashed to [−soft_cap, +soft_cap].
tanh is smooth and differentiable — no gradient killed at the boundary.
Why tanh and not just clipping? Hard clipping sets gradient = 0 beyond the cap, killing learning. tanh has a smooth gradient everywhere — large logits are suppressed but still trainable.
