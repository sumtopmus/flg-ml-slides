KV Cache & Arithmetic Intensity
The problem during inference
At every new token, attention needs K and V for all previous tokens.

Without caching: recompute K = xWₖ and V = xWᵥ for every prior token on every step. Cost grows as O(n²).
KV Cache solution
Save K and V for all past tokens as they're computed.

Each new token: compute K, V only for itself, then append to cache and read the rest.

Total memory accesses reduced drastically — no recomputation.
FLOPs / bytes
Arithmetic intensity — ratio of compute to memory reads. KV cache makes attention memory-bound: we read more than we compute.
2 · n_layers · d
KV cache size per token (K + V, all layers). Grows linearly with sequence length.
Bottleneck
Long context = huge cache. Memory becomes the constraint, not FLOPs. Motivates MQA and GQA.
