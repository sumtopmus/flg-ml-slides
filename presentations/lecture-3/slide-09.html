Attention Head Variants
Classic
Multi-Head Attention (MHA)
Q₁K₁V₁ Q₂K₂V₂ Q₃K₃V₃
Each head has its own K and V projections. Highest quality, highest memory cost.
Fast
Multi-Query Attention (MQA)
Q₁ Q₂ Q₃ K V (shared)
One shared K and V for all query heads. Massively reduces KV cache size. Small quality drop.
Modern default
Grouped Query Attention (GQA)
Q₁Q₂ KV Q₃Q₄ KV
Groups of query heads share one KV pair. Best of both worlds — quality close to MHA, memory close to MQA.
KV Cache
During inference, K and V are cached across steps. Fewer KV heads = smaller cache = faster generation and lower memory. Critical at long contexts.
Sparse / Sliding Window Attention
For very long sequences, each token only attends to a local window of nearby tokens. Reduces attention from O(n²) to O(n·w) where w is window size.
